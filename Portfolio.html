Demonstration XML-Parser

Es hat bereits mehrere Fälle gegeben, in denen jemand eine XML-Datei, die dazu benötigt wird, um unsere Software für die Brennöfen aktuell zu halten, bearbeitet hat und dabei beispielsweise ein falsches Datumformat oder eine ungültige Bezeichnung eines Tags verwendet hat, und das hat letztendlich den ganzen Updateprozess der Software lahmgelegt. Also habe ich jetzt den Auftrag bekommen, einen Parser zu schreiben, der erst überprüft, ob das XML-File tatsächlich valide ist oder ob sich darin noch Fehler befinden, bevor man dieses hochladen kann. Wir haben uns dazu entschieden, dass ich dies in Python mache und die Eingabe der XML-Datei sowie die Fehlerausgabe, oder sollte es keine Fehler mehr haben die Meldung, dass das File valide ist, über die Command Line erfolgen soll.

Codeausschnitt XML-Parser

Hier ist ein Ausschnitt des Codes für das Parsen des XML-Files. Der Ausschnitt zeigt, wie jweils das Kind-Element aus dem ChildEintrag-tag geholt wird und überprüft wird, ob die Bezeichnung gültig und an der richtigen Stelle ist, sowie ob das Kind-Element weitere Kind-Elemente besitzt. Falls etwas nicht den Vorgaben entsprechen sollte, so wird direkt eine entsprechende Fehlermeldung ausgegeben, dazu werden die tatsächlichen Tag-Namen aus dem XML genommen, selbst wenn diese falsch sein sollten, und die Anzahl der Fehler wird angepasst. Der Fehlerzähler wird unter anderem benötigt, um am Ende überprüfen zu können, ob das File nun valide ist oder es nicht gültig ist.

Konzept XML-Parser

Auf dem vollständigen Konzept, dem dieser Ausschnitt entstammt, basiert der Code und somit auch die ganze Applikation. Das Konzept ist in Deutsch gehalten, während der Code so weit wie möglich auf Englisch ist. "Eintrag-Tag" aus dem Konzept würde beispielsweise dem "ChildEintrag"-Element aus dem Code entsprechen.
